# Toqan: Building a Production-Ready SQL Data Analyst Agent (2023)

https://www.youtube.com/watch?v=gLuSbnMctfs

## Short Summary

Toqan's journey in deploying their data analyst agent that translates natural language questions into SQL queries, focusing on making it production-ready for hundreds of users. The team identified and solved four key challenges: implementing deterministic workflows for behavioral issues, leveraging domain experts for setup and monitoring, building system resilience for handling diverse inputs, and optimizing tools to simplify agent work. Their approach balanced automated capabilities with controlled workflows, resulting in a robust and widely adopted solution.

## Long Summary

This case study explores Toqan's experience in deploying a specialized AI agent that functions as a data analyst, capable of converting natural language questions into SQL queries. The presentation was delivered by two key team members: Yannis and Don, a machine learning engineer, who shared their journey of transforming a promising but initially unstable agent into a production-ready system serving hundreds of users.

### Core System Overview

The Toqan data analyst agent is designed to:

• Accept natural language questions from users
• Translate these questions into SQL queries
• Execute the queries against databases
• Return and visualize results
• Handle error recovery and query refinement
### Key Challenges and Solutions

The team identified four fundamental areas that required careful attention for successful production deployment:

1. Deterministic Workflows for Behavioral Problems
The initial implementation faced challenges with the unpredictable nature of LLM responses. Users would often ask vague questions, and the agent would attempt to provide answers even when it lacked sufficient context or understanding. To address this, they implemented:

• A pre-processing step to evaluate questions in isolation before attempting to solve them
• Hard-coded validation checks for required behaviors
• SQL query validation before execution to catch common errors
• This approach helped create a more consistent user experience and reduced unnecessary system load
2. Expert Integration and Monitoring
The team learned the importance of treating their agent as a data product requiring rigorous testing and expert oversight. They implemented:

• Pipelines and UIs for data experts to update documentation
• Continuous feedback loops with domain experts
• Tools for experts to optimize documentation specifically for agent consumption
• This allowed for rapid development cycles and specialized optimization of the agent for specific use cases
3. System Resilience
As users began interacting with the system in unexpected ways, the team needed to build robust handling of edge cases. They implemented:

• A balance between hard limits (strict rules about when the agent must stop) and soft limits (nudging the agent in the right direction)
• Mechanisms to prevent infinite loops while preserving creative problem-solving capabilities
• Clear boundaries for acceptable queries and inputs
4. Tool Optimization
The team found that optimizing individual tools within the agent framework was crucial for overall system performance:

• Enhanced schema tool to better determine relevant information
• Improved SQL execution tool with built-in error handling
• Focused on reducing context clutter and unnecessary agent cycles
• Implementation of timeouts and access controls
### Technical Implementation Details

The system architecture includes several notable features:

• Custom-built framework rather than using existing agent frameworks
• Multiple LLM models used for different components, with model selection based on task complexity and cost considerations
• More powerful models for core analysis tasks, lighter models for simpler operations
• Implementation of role-based access control for database connections
• Secret management systems for handling database credentials
• Reflection tools for error handling and query refinement
### Security and Access Control

The team implemented careful controls around database access:

• Case-by-case management of database connections
• Limited access credentials to prevent system-wide impacts
• Timeout mechanisms for query execution
• Strict requirements for credential management
### Lessons Learned

Some key takeaways from their experience include:

• "AI-first doesn't mean AI-only" - the importance of balancing automated capabilities with controlled workflows
• The value of deterministic checks in an otherwise probabilistic system
• The necessity of expert involvement in ongoing system development and monitoring
• The importance of building resilient systems that can handle unexpected user behaviors
### Results and Impact

The system successfully scaled to serve hundreds of users, demonstrating the effectiveness of their approach. The team's focus on balancing automation with control, and their emphasis on expert involvement and system resilience, proved crucial for production success.

### Future Considerations

While the current implementation uses a single primary agent with various tools, the architecture is designed to potentially support multi-agent configurations in the future. The team continues to evaluate new models and frameworks while maintaining their focus on precision and reliability, particularly given the high cost of errors in data analysis contexts.
